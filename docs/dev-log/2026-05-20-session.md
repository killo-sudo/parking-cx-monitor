## 2. 근본 원인

### 2-1. 워크플로우 git add 오류
```yaml
# 기존 (문제)
git add docs/data.json docs/app_info.json docs/datalab_trends.json data/ -f 2>/dev/null || true

# -f 플래그가 pathspec 뒤에 위치 → git이 "-f"를 파일명으로 해석할 수 있음
# 2>/dev/null || true 로 에러 묵음 처리 → 스테이징 실패가 보이지 않았음
```

### 2-2. Sheets 병합 범위 제한
```python
# 기존: 리뷰 타입만 Sheets에서 병합
_REVIEW_TYPES = {"appstore", "ios_appstore"}
if src not in _REVIEW_TYPES:
    continue
```
→ 뉴스·앱 업데이트 등 비리뷰 항목은 Sheets에서 가져오지 않아 SQLite에만 의존

### 2-3. 날짜 불명 리뷰에 오늘 날짜 fallback
```python
# Android
pub_dt = r.get("at", datetime.now())   # at 없으면 오늘 날짜
if pub_dt.year < 2020:
    pub_dt = datetime.now()            # 파싱 오류도 오늘 날짜

# iOS
pub_str = updated[:10] if updated else datetime.now().strftime("%Y-%m-%d")
if pub_str[:4] < "2020":
    pub_str = datetime.now().strftime("%Y-%m-%d")
```
→ Sheets A열(날짜)에 실제 작성일이 아닌 수집일이 기록됨

```yaml
# 수정 후
- name: Commit updated data files
  run: |
    git config user.name  "github-actions[bot]"
    git config user.email "github-actions[bot]@users.noreply.github.com"
    git add -f docs/data.json                                    # -f를 앞으로
    git add -f docs/app_info.json docs/datalab_trends.json 2>/dev/null || true
    git add data/ 2>/dev/null || true
    echo "=== git status ==="
    git status --short
    echo "=== staged diff stat ==="
    git diff --cached --stat
    if ! git diff --cached --quiet; then
      git commit -m "chore: update data.json + app_info.json [skip ci]"
      git pull --rebase --autostash origin main
      git push
    else
      echo "Nothing to commit — data.json unchanged"
    fi
```
